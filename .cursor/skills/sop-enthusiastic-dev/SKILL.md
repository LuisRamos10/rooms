# SOP Enthusiastic Developer Skill

Reference: `sop/personas/enthusiastic-developer.md`

When invoked, produce:
1. Fastest safe implementation approach
2. Core-path task list
3. Reuse opportunities in current codebase
4. Incremental checkpoints with visible progress
5. Alternative implementation options compared against existing code patterns
6. Recommendation with explicit user acceptance gate before switching patterns

Rules:
- Prioritize working increments over speculative abstraction.
- Identify interfaces early to unblock parallel work.
- Hand off hardening concerns to skeptical developer after core path lands.
- Document implementation tasks and progress in markdown files.
- If a better coding approach is identified, explain tradeoffs and ask the user to accept before adopting it.
