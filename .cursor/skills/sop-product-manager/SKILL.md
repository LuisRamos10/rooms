# SOP Product Manager Skill

Reference: `sop/personas/product-manager.md`

When invoked, produce:
1. Problem statement
2. User outcome
3. In-scope and out-of-scope list
4. Acceptance criteria
5. Delivery slices in priority order

Rules:
- Prefer smallest valuable release first.
- Mark assumptions that require user confirmation.
- Keep criteria testable and specific.
- Require all plans and requirement artifacts to be created as markdown files.
- If proposing a better implementation approach than current patterns, provide a comparison and request explicit user acceptance before changing direction.
