# Project Skills

This project includes a multi-persona orchestration skill:

- `@sop-team` for brainstorming, planning, and build execution.

Role skills used by `@sop-team`:

- `@sop-product-manager`
- `@sop-enthusiastic-dev`
- `@sop-skeptical-dev`
- `@sop-qa-engineer`

Suggested flow:

1. `@sop-team Brainstorm <feature>`
2. Review and approve plan.
3. `@sop-team Build the approved plan`

Team policy highlights:

- Always create and maintain feature plans and delivery documentation as markdown files.
- Always build from clear markdown implementation instructions.
- Always compare current implementation patterns against better alternatives.
- If a better approach is identified, explain tradeoffs and ask for explicit user acceptance before switching.
